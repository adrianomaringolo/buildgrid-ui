# Button

The `Button` component is a versatile, customizable button designed to meet various design and interaction needs. It includes support for accessibility, multiple styles (variants), sizes, loading states, and more.

## Features

- **Variants:** Multiple pre-designed styles (default, destructive, outline, etc.).
- **Sizes:** Flexible size options ranging from small to extra-large.
- **Loading State:** Built-in loading spinner for asynchronous actions.
- **Customizability:** Supports custom classes and `asChild` for rendering as different HTML elements.

## Usage

Import the `Button` component and use it like any other React component. Here’s a basic example:

<CodeBox code={`<Button>Click Me</Button>`}>
 <Button>Click Me</Button>
</CodeBox>

## With a Loading State

The `Button` component supports a loading state using the `isLoading` prop.

<CodeBox code={`<Button isLoading>Submitting...</Button>`}>
	<Button isLoading>Submitting...</Button>
</CodeBox>

The `variant` prop controls the button’s appearance. The following variants are available:

| Variant      | Description                                   | Example                              |
| ------------ | --------------------------------------------- | ------------------------------------ |
| `default`    | Primary button for main actions.             | `<Button>Default</Button>`          |
| `destructive`| Indicates a destructive or negative action.  | `<Button variant="destructive">Delete</Button>` |
| `outline`    | Outlined button with a minimal design.       | `<Button variant="outline">Outline</Button>` |
| `secondary`  | Secondary actions with softer visuals.       | `<Button variant="secondary">Secondary</Button>` |
| `ghost`      | Button with a transparent background.        | `<Button variant="ghost">Ghost</Button>` |
| `link`       | Styled as a hyperlink.                      | `<Button variant="link">Link</Button>` |



## Sizes

The `size` prop adjusts the button's dimensions. The following sizes are available:

| Size    | Description                     | Example                        |
| ------- | ------------------------------- | ------------------------------ |
| `default`| Standard size.                 | `<Button size="default">Default</Button>` |
| `sm`    | Small button.                   | `<Button size="sm">Small</Button>` |
| `lg`    | Large button.                   | `<Button size="lg">Large</Button>` |
| `xl`    | Extra-large button.             | `<Button size="xl">Extra Large</Button>` |
| `icon`  | Square button for icons only.   | `<Button size="icon">+</Button>` |

## Accessibility

- The button uses `focus-visible` styles to provide clear focus indication.
- Loading state disables interaction to prevent multiple submissions.
- ARIA roles are applied correctly when used in different contexts.

## Example

```tsx
<Button variant="primary" size="lg" isLoading>
  Loading...
</Button>
```
